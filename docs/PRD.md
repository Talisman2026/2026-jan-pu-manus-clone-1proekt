# PRD: AgentFlow MVP

**Версия:** 2.0 MVP | **Дата:** 2026-03-11  
**Статус:** Ready for development

---

## 1. Что строим

**AgentFlow MVP** — веб-приложение где зарегистрированный пользователь может запустить автономного AI-агента для выполнения сложной задачи и получить результат в виде файла.

**Пользовательский путь:**
```
Регистрация/логин
→ Вводит OpenAI API key (один раз, хранится в браузере)
→ Описывает задачу текстом
→ Видит pre-run estimation (шаги, примерное время)
→ Подтверждает запуск с budget cap
→ Наблюдает лог выполнения в реальном времени
→ Скачивает файл результата
```

---

## 2. Цель MVP

Проверить одну гипотезу: **пользователь готов использовать автономного агента для реальных задач.**

Для этого достаточно:
- Агент реально выполняет задачи (research, analysis, data collection)
- Пользователь видит что происходит (лог)
- Получает результат в виде файла

Что НЕ нужно проверять на MVP:
- Монетизация (нет Stripe)
- Масштаб (одна задача за раз)
- Мобильный UI

---

## 3. Функции MVP

### F1: Авторизация
- Регистрация по email + пароль
- Логин / логаут
- JWT сессии

### F2: Настройка API ключа
- Пользователь вводит свой OpenAI API key в Settings
- Ключ шифруется (AES-GCM) и хранится в IndexedDB браузера
- Ключ никогда не сохраняется на сервере

### F3: Создание задачи
- Форма: описание задачи (textarea)
- Pre-run estimation: примерное количество шагов и время
- Budget cap: пользователь устанавливает лимит в $ перед запуском

### F4: Выполнение задачи
- Агент запускается в изолированном E2B sandbox
- Инструменты: web_search (Firecrawl), scrape_url, run_python, write_file
- Одна задача за раз на пользователя

### F5: Observability
- Лог выполнения в реальном времени (polling каждые 2 сек)
- Каждый шаг: инструмент + описание + статус
- Budget progress bar
- Статусы: estimating → running → completed / failed / paused

### F6: Результат
- Скачать файл результата (.md, .xlsx, .csv, .html, .txt)
- Краткое summary в UI
- История задач (список с статусами)

---

## 4. Что НЕ входит в MVP

- Stripe / оплата / тарифные планы
- Multi-agent / параллельные задачи
- Sharing результатов
- Email уведомления
- Mobile UI
- Self-host
- API для разработчиков

---

## 5. Success criteria

MVP считается успешным если:
- Агент успешно выполняет ≥ 5 разных типов задач
- Пользователь понимает что происходит без объяснений
- Время от "запустить" до первого результата ≤ 15 мин
